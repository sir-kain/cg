QUICK GUIDE

Quick guide to set up most basic for active rubytabs. The other section will provide detailed options.

1. HOSTING : upload ruby folder to your hosting, this folder includes main script, css and other parts of rubytabs.

  • ruby

Other folders included :

  • documentation | documentation used in rubytabs
  • templates | including the entire pure HTML templates
  • libs | including components used in theme


2. RUBYTABS FILES : insert main script, css of rubytabs on site :

<!-- Main css of rubytabs, including styles, skins -->
<link rel="stylesheet" href="ruby/rubytabs.css">

<!-- jQuery 1.9+ required -->
<script src="//ajax.googleapis.com/ajax/libs/jquery/1.12.4/jquery.min.js"></script>

<!-- Main script of rubytabs -->
<script src="ruby/rubytabs.js"></script>

<!-- CSS effects (CSS One - Two - Four effects) in rubytabs -->
<link rel="stylesheet" href="ruby/rubyanimate.js">


3. MARKUP HTML : insert rubytabs html on site :

  • Namespace of RubyTabs used to distinguish with other plugins, no change. rt01
  • Class .rt01 → core css of rubytabs.
<div class="rt01">
    <!-- Slide 1 -->
    <div>
        <!-- Slide title -->
        <div class="rt01pagitem">Slide title</div>

        <!-- Slide content -->
        <p>Lorem ipsum dolor sit amet, consectetur adipisicing elit...</p>
    </div>

    <!-- Slide others same Slide 1 -->
    <div>...</div>
</div>


4. SETTING OPTIONS : initialize the rubytabs

HTML5 DATA
  • Setting options directly on the markup by HTML5 data
<div class="rt01"
    data-tabs='{
        "fx"    : "cssOne",
        "speed" : 800,
        "pag"   : { "align" : "center" }
    }'>
    ...
</div>
JAVASCRIPT

Setting options by javascript like other jQuery plugins

<script>
    jQuery(document).ready(function($) {
        $(".rt01").rubytabs({
            "fx"     : "cssOne",
            "speed"  : 800,
            "pag"    : { "align" : "center" }
        });
    });
</script>
With a few simple steps, rubytabs ready to work!

TABLE OPTIONS

General options
Variable Default value Description
fx "line" Setup main effect.
cssOne "roDeal" Setup CSS One effect
cssTwoOut "slideShortDownOut" Setup name for Out effect of CSS Two effects
cssTwoIn "pushSoftIn" Setup name for In effect of CSS Two effects
cssFourPrevOut "roEdgeRightOut" Setup name for CSS Four effect when swap to previous slide out
cssFourPrevIn "roEdgeLeftIn" Setup name for CSS Four effect when swap to previous slide in
cssFourNextOut "roEdgeLeftOut" Setup name for CSS Four effect when swap to next slide out
cssFourNextIn "roEdgeRightIn" Setup name for CSS Four effect when swap to next slide in
fxEasing "easeOutCubic" Easing of main effect.
cssEasing null Easing of effect in CSS One, Two, Four effects.
speed 400 Duration of the effect. Unit millisecond.
margin 30 Distance between slides. Unit is px and supports distances left and right. Example:
  • "margin": [10, 20] → 10px is left distance, 20px is right distance
  • "margin": 30 → 30px is left and right distance
idBegin 0 Set the slide will appear as rubytabs initialization, ID of slides started with 0. List of value option:
  • "begin": begin position, equivalent to idBegin = 0
  • "center": center position. If the total number of slides is an even number, then the position will be located near the left side. Example: Num of slide is 6, value of idBegin is 2
  • "centerRight": similar "center" value, but located near the right side. Example: Num of slide is 6, value of idBegin is 3
  • "end": end position
  • 0 1 2 ...: specific value of the ID slide
showBy "all" Set rubytabs initialized only on the specified device. List of value: "desktop", "mobile" and "all".
showFrom 0 Set rubytabs will appear in range width of the site. Range-width have 2 values "min-width" and "max-width"(optional).
     
isAutoInit true Option exclusively for HTML5 data. RubyTabs is automatically initialized after markup have loaded.
isSlideshow false Turn on/off slideshow.
isNav false Turn on/off navigation tabs.
isPag true Turn on/off pagination(tablist) tabs.
isSwipe true Turn on/off swipe gestures on rubytabs.
isKeyboard false Turn on/off keyboard navigation, left/right arrow on keyboard to go prev/next slide.
isDeeplinking false Turn on/off deep-linkinging features.
isCookie false Turn on/off cookie features.
name null The proper name for each rubytabs in the site. Used to link components markup outside with main markup of rubytabs.
nameViewport "viewport" Set the name-class of the component "viewport".
nameCanvas "canvas" Set the name-class of the component "canvas".
nameSlide "slide" Set the name-class of the components "slides".
nameDataLazy "src" Name to be replaced in "data-src" on <img> tag using image lazyload function.

Options in load object
Variable Default value Description
preload 1 The number of slides(tabs) preloaded before the rubytabs appears.
amountEachLoad 2 The number next slides will load after rubytabs appears, the slides will load in parallel.
isLazy true Turn on/off only download the ImageLazy of selected slide.

Options in pag object
Variable Default value Description
type "tabs" Type of pagination(tablist). List of value : "thumbnail", "tabs", "bullet", "list".
width null Setup fixed width for pagItem. By default, the pagItem will get largest width in the pagItems.
height null Setup fixed height for pagItem. By default, the pagItem will get largest height in the pagItems.
direction "hor" Setup the direction of pagination. List of value: "hor", "ver".
position "begin" Setup the position of pagination compared to the content tabs. List of value: "begin", "end".
align "begin" Align of pagItems compared to pagination. List of value: "begin", "center", "end", "justify".
speed 300 Time to transition of pagItem current automatically move to the center position.
ease "easeOutCubic" Easing transition of pagItem current automatically move to the center position.
sizeAuto "full" Setup width(horizontal direction) / height(vertical direction) of pagination compared to width/height of rubytabs.
  • "null": size of pagination depends on css.
  • "self": size pagination is equal to size all pagItems combined.
  • "full": size pagination is equal to size content of rubytabs.
typeSizeItem "self" Get the size of each pagItem compared to the size of other pagItems. List of value: "self", "min", "max".
moreClass null Adding classes to the pagination markup.
widthMinToHor 0 Minimum width of RubyTabs to switch to horizontal direction.
rangeMinToHor 0 Minimum width of browser-document to switch to horizontal direction.
isItemCurCenterWhenTap true Turn on/off the current pagItem automatically moves into the center position of pagination.
isArrow true Turn on/off navigation next/previous of pagItem.
isTapOnArrow true Add tap event on navigation next/previous
isMark false Turn on/off mark of pagItem current. Supported animation.
sizeMarkTo "border" The size of mark depends on the size (can include "padding", "margin"..) pagItem current. List of value: "self", "padding", "border", "margin".

Options in swipe object
Variable Default value Description
isBody false Turn on/off swipe gestures in body content of tabs.
isAutoOnPag true Turn on/off auto swipe gestures on pagination when the total size of pagItems larger size pagination.

POSITIONS

With a combination of 3 options: "direction", "position", "align" → to create 14 different position of tablists.

  • Direction : "hor", "ver"
  • Position : "begin", "end"
  • Align : "begin", "center", "end", "justify"

DIRECTION

There are 2 directions in RubyTabs: horizontal and vertical, corresponding 2 keywords "hor" and "ver".

Horizontal Direction Vertical Direction
HTML5 DATA 
<!-- Horizontal Direction setup -->
<div class="rt01" data-tabs='{ "pag": { "direction": "hor" } }'> ... </div

<!-- Vertical Direction setup -->
<div class="rt01" data-tabs='{ "pag": { "direction": "ver" } }'> ... </div
JAVASCRIPT 
// Horizontal Direction setup
$(".rt01").rubytabs({ "pag" : { "direction": "hor" } });

// Vertical Direction setup
$(".rt01").rubytabs({ "pag" : { "direction": "ver" } });

POSITION

Each horizontal and vertical directions are included 2 position options: "begin" and "end".

Begin Position End Position
HTML5 DATA 
<!-- Begin Position setup -->
<div class="rt01" data-tabs='{ "pag": { "position": "begin" } }'> ... </div

<!-- End Position setup -->
<div class="rt01" data-tabs='{ "pag": { "position": "end" } }'> ... </div
JAVASCRIPT 
// Begin Position setup
$(".rt01").rubytabs({ "pag" : { "position": "begin" } });

// End Position setup
$(".rt01").rubytabs({ "pag" : { "position": "end" } });

ALIGN

In each position options, cluster tablists than rubytabs have 4 align options: "begin", "center", "end" and "justify".
Vertical directions options not supported "justify" align options.
If the length clusters tablists larger body tabs, rubytabs automatically switch on "begin" align options.

LAZYLOAD

RubyTabs supported: lazy loading each slide, lazy loading nearby slide current, lazy loading image - video - iframe.

SMART LOADING

RubyTabs initialize and display immediately after the first slide downloaded, no need to waitting for all the images loaded.

IMAGE LAZY

Images will be downloaded until slide parent is selected.
RubyTabs will control the size of image in responsive mode.

Image Lazyload 
<!-- Image normal -->
<img src="/img.jpg" alt="image alt">

<!-- Image lazyload using link tag -->
<a class="rt01img" href="/img.jpg">image alt</a>

YOUTUBE - VIMEO LAZY

Video is hidden under the image placeholder and doesn't load heavy iframe until required

Video Lazyload 
<!-- Insert video link to image lazyload or image background -->
<a class="rt01img" href="imgs/youtube-preview.jpg" data-video-link="https://www.youtube.com/watch?v=...">Youtube preview alt</a>

<a class="rt01img" href="imgs/vimeo-preview.jpg" data-video-link="https://vimeo.com/...">Vimeo preview alt</a>

IFRAME LAZY

Iframe (map, video iframe...) will be downloaded until slide parent is selected.

Iframe Lazyload 
<!-- Map iframe normal -->
<iframe src="https://www.google.com/maps/..." width="600" height="450" frameborder="0" style="border:0" allowfullscreen></iframe>

<!-- Map iframe lazyload using link tag -->
<a class="rt01iframe" href="https://www.google.com/maps/..." width="600" height="450" frameborder="0" style="border:0" allowfullscreen></a>

SLIDESHOW

Options in slideshow object
Variable Default value Description
delay 8000 Setup wait time to move next slide of rubytabs in slideshow. Unit is millisecond.
timer "arc" Setup timer of slideshow. List of value: "arc", "line".
isAutoRun true Turn on/off auto-running slideshow after initialization rubytabs.
isPlayPause true Turn on/off "playpause" component in slideshow.
isTimer true Turn on/off "timer" component in slideshow.
isHoverPause false Turn on/off features mouse over on rubytabs, slideshow automatically pause.
isRunInto true Turn on/off features only slideshow when rubytabs displayed on areas visible of browser.
isRandom false Turn on/off auto switch to random slide in slideshow.

BASIC SETUP

  • Turn on slideshow by setup options "isSlideshow" : true. Default options is "isSlideshow" : false.
  • The time to switch between slides are setted with "delay" options in "slideshow" object options. Unit is milliseconds(ms).
  • Add style for timer by "class-name" to head markup rubytabs → i.e. ".rt01timer-arcTop"
HTML5 DATA 
<!-- Combine add style for timer and turn on slideshow -->
<div class="rt01 rt01timer-arcTop" data-tabs='{
    "isSlideshow" : true,
    "slideshow"   : { "delay": 5000 },
    ...
}'> ... </div>
JAVASCRIPT 
<!-- Setup style for timer -->
<div class="rt01 rt01timer-arcTop"> ... </div>
// Setup option for slideshow
$(".rt01").rubytabs({
    "isSlideshow" : true,
    "slideshow"   : { "delay" : 5000 },
    ...
});

Customization of the "timerArc" object

The options in "arc" object used to customize color, thickness, size of circle timer.
Arc timer included "Inner" circle and "Outer" circle. Each circle are 4 options in "arc" object.
"Inner" circle used to display the time loop in slideshow.
"Outer" circle used to decorate or background.

Detailed description of the class is added:

@ Markup rubytabs

  • class .rt01ready: will be added when first slide of rubytabs complete loaded, and rubytabs began to appear in browser.
  • class .rt01layout-dot : type effect of rubytabs. Effect CSS One, Two, Four and fade are "dot" type, also "line" effect is "line" type.
  • class .rt01fx-cssOne : current name of effect.
  • class .rt01height-auto : type height of rubytabs, include 2 option - "height-auto" and "height-fixed".
  • class .rt01transition : to recognize the browser supports CSS3 Transition. No supported, is "no-transition".
  • class .rt01pag-hor : will be added when pagination turn on. Direction of pagination, include 2 options: "pag-hor" for horizontal direction, "pag-ver" for vertical direction.
  • class .rt01pag-begin : will be added when pagination turn on. Position of pagination, include 2 options: "pag-begin" and "pag-end"

@ Markup viewportName
Container all slide, often used to css of slide area


@ Markup canvas
This markup only used for rubytabs system, don"t css on this markup.


@ Markup slide

  • class .rt01ready: will be added when all images on slide are complete loaded
  • class .rt01cur: will be added when this markup is current slide to appear rubytabs

@ Markup pag-head

  • class .rt01tabs: type of pagination, include "tabs" - "bullet" - "thumbnail"
  • class .rt01pag-hor   .rt01pag-begin: similar class on @ markup rubytabs, direction and position of pagination. 2 classes are added to ensure css pagination outside the rubytabs

@ Markup pag-inner
This markup only used for rubytabs system, don"t css on this markup.


@ Markup pag-item

  • class .rt01cur: will be added when this markup is current pagination-item selected.

@ Markup cap-inner
This markup only used for rubytabs system, don"t css on this markup.


@ Markup playpause
Button play/pause in slideshow

  • class .rt01actived: will be added when users switch to stop state

@ Markup timer-head

  • class .rt01timer-line: type of timer, include "timer-arc" - "timer-line"

@ Markup timer-item
This markup described the current time taking place in the slideshow.
Depending on type of timer, tagName timer-item change as well:

  • "canvas" tagName for timer-arc
  • "div" tagname for timer-line

Outside Markup

+ Changed - version 1.4:
data-tabs='{ "name": "abc" } to data-rubytabs-markup="abc" on markup outside object.

The components: navigation, title tabs(pagination), timer... can set up markup in any location on site.
Support freedom of layout, adding "class-name" on the components.

Setup of outside markup:

  • First, naming rubytabs with any name(no space) via name option.

API

Public methods

/* All public medthods can be called
$(".rt01").rubytabs("next");
$(".rt01").rubytabs("goto", 3);

/* You can be create instance rubytabs data and many call quickly */
var ruby = $(".rt01").data("rubytabs");

ruby.next(num);                     // Next to number of slide. Default "num" = 1
ruby.prev(num);                     // Previous to number of slide. Default "num" = 1
ruby.first();                       // Goto first slide
ruby.last();                        // Goto last slide
ruby.goto(id);                      // Goto slide id number, begin 0

ruby.update(opts [, isNoRefresh]);  // Update options and refresh tabs
                                    // Example: ruby.udpate({ speed: 1200, fx: "fade" });
ruby.refresh();                     // Refresh tabs
ruby.destroy([isDelete]);           // Removes all events and clears all data on tabs

ruby.play();                        // Play slideshow
ruby.pause();                       // Pause slideshow
ruby.stop();                        // Stop slideshow

ruby.addSlide(obj [, index]);       // Add new slide. If "index" is empty, the new slide will auto add at the end
ruby.removeSlide([index]);          // Remove the slide at "index" position.
ruby.orderSlide(order);             // Reorder slides. Varible "order" can be object or array
                                    // Example: ruby.orderSlide({ 0: 1, 3: 0});

ruby.swipeEvent(objName);           // Register or remove swipe event on object
                                    // "onBody"  : register swipe event on body tabs
                                    // "offBody" : remove swipe event on body tabs
                                    // "onPag"   : register swipe event on pagination
                                    // "offPag"  : remove swipe event on pagination

TABLE OPTIONS

Variable Default value Description
optionsPlus "tabs" Many commonly used options in slider, these options will override the default options. View list options below for "slider" value.
height null Height of the slider. By default, it depends on the height image-background per slide.
width null Enable responsive and settings width-begin(or maximum) of the slider.
widthSlide 1 Width of center slide compared to width of slider.
  • Value in range [0-1] is the percentage(%) of slide compared to the slider.
  • Value greater than 1 is unit "px".
  • Supports "range" features.
widthRange null
  • Set width of image-background in the range width of the page.
  • Supports multiple range.
  • Structure: { "media" : [width slide, from, to] }
padding 0
  • Surrounding areas of image-background to slide in th range width of the page.
  • Supports multiple range.
  • Structure: { "padding" : [value, from, to] }
direction "hor" Direction swipe gestures on the body content.
There are 2 directions: "hor" & ver.
imagePosition "center" Position and size of imageback to fit the viewport slider. List of value: "center", "fill", "fit", "stretch", "tile".
offsetBy null In Fullscreen mode, the heigth of slider equal to the height of window browser minus the height of offset objects.
isCenter true Turn on/off layout center in effect "line".
isCap false Turn on/off caption each slide.
isFullscreen false Turn on/off the height of slider equal to the height of window browser.

The default size of the slider:
+ Width of slider depend on the object containing it.
+ Height of slider depend on each height of image-background per slide.

Responsive slider just installed the original(or maxium) width of the slider. Height/width of image-background will change ratio follow width of the slider.

    "width" : 940,

});

CENTER SLIDER

In "line" effect, to layout center should set width of middle slide smaller width of slider through "widthSlide" option.
"widthSlide" : width-slide

Center Layout
HTML5 DATA
<!-- Value in range [0-1] is the percentage(%) of slide compared to the slider -->
<div class="rt01" data-tabs='{ "fx": "line", "widthSlide": 0.8, ... }'> ... </div>

<!-- Value greater than 1 is unit "px" -->
<div class="rt01" data-tabs='{ "fx": "line", "widthSlide": 748, ... }'> ... </div>
JAVASCRIPT
$(".rt01").rubytabs({
    "fx"         : "line",
    "widthSlide" : 0.8,
    ...
});

Special: "widthSlide" option also supports "media" features - multi width-values in different width of the page.
"widthSlide" : [width-slide, from, to]

Example: Setup center slider vi width middle slide change over width of the page.

  • Any width page: width center slide = 70% width slider
  • Width page range [0 - 480px]: priority width center slide = 300px
  • Width page range [1200px - 3000px]: priority width center slide = 1156px
HTML5 DATA
<div class="rt01" data-tabs='{
    "fx"         : "line",
    "widthSlide" : [ 0.7, [300, 0, 480], [1156, 1200, 3000] ],
    ...
}'> ... </div>
JAVASCRIPT
$(".rt01").rubytabs({
        "fx"         : "line",
        "widthSlide" : [ 0.7, [300, 0, 480], [1156, 1200, 3000] ],
        ...
    });
